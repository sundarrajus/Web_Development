CSS Z-Index

1. Default Stacking — No z-index

Without z-index, elements stack in HTML order — the last element in the HTML appears on top.

1st in HTML
2nd in HTML
3rd (on top)

2. z-index — Control the Stack Order

z-index controls which element appears on top. Higher number = on top.

Important: z-index only works on elements that have a position value set (relative, absolute, fixed, or sticky).

In the example below: z-index order is 10 → 100 → 101 → 102

z:100
z:101
z:10 (behind!)
z:102 (top!)

3. Real Use — Tooltip with z-index

Tooltips appear above other content. We use z-index: 999 to ensure they're always on top.

I appear above everything! (z-index: 999)
Tooltip for button 2

4. Real Use — Modal/Popup Overlay

Modals need a dark overlay on top of the page (high z-index), and the modal box even higher.

overlay: z-index: 10 → modal box: z-index: 20

I appear on top of the overlay!

5. Key Rules to Remember

  • z-index only works when position is set (not static)
  • Higher z-index = appears in front
  • Negative z-index = goes behind other elements
  • Common values: tooltips 999, modals 1000, fixed headers 100
  • Without setting position, z-index has no effect!
